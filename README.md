# M-s-30ty-Birthday
by an idiot who wants to show her love desperately


## 25.12
确定了创意怎么实现和大致骨架，本质上这应该是个推理游戏，来源于我去年玩的大学生登山失踪事件那个游戏，但是我觉得推理难度如果太大 m 应该做不出来，我又不能给他写个攻略，所以难度简化了。

原来本来应该至少有文本阅读解密，base64、音频/图片倒放，文字反色，凯撒密码等形式，我觉得学计算机应该不会太难解答，但考虑到多重因素还是算了。

是一个网页形式，因为现在这个情况给他寄东西必然不会收的，只能是以线上的形式。主题的话又要避免道德绑架，又要用心，我觉得聚焦对他个人的探索比较合适。

准备采用童话的文风和设计，本身好制作（有主题），没那么此地无银。

## 26.2
文案定稿。  
改了很多很多遍，分主题但主题之间有关联其实是非常困难的事。

要分页，设计每一页的视觉效果和主题，要把我对他一些小点的观察总结和整合成一定数量有关联的小主题，需要找到内部的关联。  
要结合童话风格，要埋进我自己的彩蛋但不能太明显。  
很多地方全靠猜，如果其实他不是我猜的那种人就尬了，但我相信我的识人能力，也相信我会喜欢什么样的人。
